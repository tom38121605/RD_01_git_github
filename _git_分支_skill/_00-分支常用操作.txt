
说明：在git命令行中，拷贝只能用鼠标，不能用ctrl + c


   
--------------------------------------------- 本地分支操作 ------------------------------------------------
 
1. 创建分支
　　git branch dev              //创建dev分支  （应从当前分支master创建出一个一模一样的另一个分支dev）

1. 切换到分支
　　git checkout dev           //切换到dev分支

1. 创建并切换到分支
　　git checkout -b dev      //创建dev分支，并切换到dev分支  （相当于把上面2句合并）    


  2. 查看本地分支
      git branch                   //查看本地分支，会列出所有的分支，当前分支前面会添加一个星号
 
  2. 查看远程分支
     git branch -r         
                                                                    
  2. 本地和远程分支
     git branch -a         

3. 合并分支   
     git merge dev                  //快M合并，因dev分支是从master分支上创建的，把master分支直接快进到dev分支的最新修改。
                                              在master分支里看不到dev分支的历史切换记录
     git merge --no-ff dev      //把dev分支合并到当前分支master上， 在master分支里保留dev的历史切换记录
 
4. 删除本地分支
   git branch -d dev       //删除本地的dev分支  //d选项只能删除已经参与了合并的分支
   git branch -D dev       //删除本地的dev分支

  

--------------------------------------------- 远程分支操作 ------------------------------------------------
 
 1. 查看远程分支
     git branch -r     
     git branch -a    

2. 在本地的master分支推送到远程的master分支

   格式： git  push   远程仓库名  本地分支名 ：  远程分支名

       git push  gitee master:master     //完整写法   
       git push  gitee master               //简略写法      

2. 把本地的devr分支推送到远程的master分支
      git push  gitee dev:master         //完整写法  
      git push  gitee master               //简略写法   

2. 推送到远程分支后，建立关联
    git push -u origin  master    //以后可以直接git push，而不用指定远程和分支

2.强制推送  （谨慎使用， 会删除其他同事的修改）
   git push --force

2. 推送所有分支到远程
   git push --all

2. 向远程推送标签
   git push --tags

3.  删除远端分支   （也可在github网址上删除）
    git push github -d develop       

  。把远端分支下载到本地  （无本地分支）
      方法1：      
      git checkout -b  dev  origin/dev                  //在本地创建一个新的分支，并将远程分支的内容下载到本地分支上 
 
     方法2： 在fork上操作
        右键分支名称  -- checkout  -- track

4.  把远端分支pull到本地 （已有本地分支）

    方法1：
       git checkout  dev
       git pull origin  dev

    方法2：
       git pull origin remote_branch_name:local_branch_name        //这个命令会将远程分支的内容直接合并到本地分支中。如本地分支不存在，则会创建一个新的本地分支

5.  在github设置默认分支： github网址 -- setting  -- 左边点选branch  -- 中间defaut下面 -- 右边 双向箭头  -- 下拉列表选择默认分支



=========================其他=============================


git push origin test:master         // 提交本地test分支作为远程的master分支 //好像只写这一句，远程的github就会自动创建一个test分支
git push origin :test                // 刚提交到远程的test将被删除，但是本地还会保存的，不用担心
删除远程仓库：$ git remote rm [name]
修改远程仓库：$ git remote set-url --push[name][newUrl]
拉取远程仓库：$ git pull [remoteName] [localBranchName]
推送远程仓库：$ git push [remoteName] [localBranchName]
删除远程分支：$ git push origin :heads/[name]


查看版本：$ git tag
创建版本：$ git tag [name]
删除版本：$ git tag -d [name]
查看远程版本：$ git tag -r
创建远程版本(本地版本push到远程)：$ git push origin [name]
删除远程版本：$ git push origin :refs/tags/[name]


添加子模块：$ git submodule add [url] [path]
如：$ git submodule add git://github.com/soberh/ui-libs.git src/main/webapp/ui-libs
初始化子模块：$ git submodule init ----只在首次检出仓库时运行一次就行
更新子模块：$ git submodule update ----每次更新或切换分支后都需要运行一下
删除子模块：（分4步走哦）
1)$ git rm --cached [path]
2) 编辑“.gitmodules”文件，将子模块的相关配置节点删除掉
3) 编辑“.git/config”文件，将子模块的相关配置节点删除掉
4) 手动删除子模块残留的目录
